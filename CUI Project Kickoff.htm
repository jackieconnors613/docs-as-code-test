CUI Kickoff

  1. Review and amend the sample project plan delivered with the CUI Template Pack. The sample project plan identifies the typical phases you must execute to deliver a successful CUI project.

  2. Gather the CUI Project team together for a kick-off. We advise having the following members:

    • Project Manager

    • Executive CUI sponsor

    • Legal or governance representative

    • IT Manager

    • End-user representative

  3. Provide the CUI project team with the “CUI For Project Teams” document supplied with the CUI Template Pack. This document gives a good overview of the reasons behind CUI.

  4. Review the CUI template file that is delivered with the CUI Template Pack and make relevant changes. Refer to the CUI Template Reference Guide for CUIConfig-V100 delivered with the CUI Template Pack. Import the TCF file and review the Schema and Policy values and settings.

  5. Consult the IT requirements to start the purchase and configuration processes. The sample project plan identifies the IT pre-requisites needed to run a successful CUI project.

  6. Determine the test group. Feedback from end-users is essential. This group will help you formulate a better policy configuration.

  7. Start the communication plan to executives, the project team, and end-users. A successful CUI project must be transparent in its objectives. A thorough communication plan can help with this. Refer to the sample end-user training material delivered with the CUI Template Pack as examples.
