CUI Template

When you open the Titus Administration Console, click the Configurations tab and select CUIConfig-V100. From here, you can examine your Schema, Schema Views, and Policies.

CUI Schema

The CUI Schema shows various fields and conditions and how the fields relate to each other. The Schema defines the classification levels with which the user interacts. It is the foundation of a Titus Configuration.

Schemas contain Fields and Values used to indicate the type of information being applied to the content. For example, the Schema Field “Dissemination” contains a list of Dissemination Control Values. You can edit these Values as required. For more information on editing Values, see the Titus Administration Console User Guide.

Schema Views

The Schema View allows you to configure the fields and values you want to appear in the Select dialog.

There are two Schema Views configured in the CUIConfig-V100 file - Default and CUI Default. The CUI Default View should be used by organizations who have a CUI only requirement. The Default View includes the CUI capabilities plus additional Export Control features. See Default Schema View on page 1 for more information.

The CUI Default Schema View contains what you need for CUI.

Policies, Actions, and Events

Policies are containers for Rules and Actions. Specifically, Rules are used to control, or conditionally control, the Actions that are run/executed when an Event occurs and any specified conditions have been met. The conditions must be evaluated as true to apply the Action that has been associated with the Rule.

The following Policy Events are supported in the CUI template:

  • Occurs when a message is sent (Titus Message Classification or Titus Classification for Microsoft Outlook)

  • Occurs before any open document is saved (Titus Classification for Microsoft Office)

  • Occurs when user changes the classification of a document (Titus Classification for Microsoft Office)

The following Action types are in the CUI template:

Body Tagging When this action is applied using the Occurs when new message is started or the Occurs when user changes classification of a message event, the Body Tagging contents are displayed in the message before the email is sent. This allows the message author to see how the Body Tag will appear to the message recipients. In order to ensure that the Body Tag has not been edited by the message author in preview mode and that it reflects the correct information, a Policy must also be created using the Occurs when a message is sent event.
Apply Advanced Header/Footer

The Apply Advanced Header/Footer Action can be used to insert the currently selected classification into the header and/or footer of the document using HTML. This Action lets you apply Field Values to the header and/or footer of the document.
Apply HTML Header/Footer The Apply HTML Header/Footer Action can be used to insert text and/or the currently selected classification into the header and/or footer of the document. This Action only applies to Microsoft Word documents.
Apply Custom X-Header This Apply Custom X-Header applies custom headers to email messages. The Titus Email Solution stores classification metadata in the X-headers of email messages. The Custom X-header Action enables you to specify additional X-headers for classification metadata. Custom X-headers are typically used for compliance with classification standards, such as the Australian Email Protective Marking Standard. They are also used for interoperability with third-party applications that require the X-header to appear in a specific format.

For more information on these Action types, see the Titus Classification for Windows Policy Authoring User Guide.

How the Classification appears in Email and Documents

Because of the CUI Policy and Actions, when you select values from the CUI Category and Dissemination Control in the Select dialog, their associated markings will display in the header and footer as per the Banner Marking: Specified Authorities column in https://www.archives.gov/cui/registry/category-marking-list, and the Marking column in https://www.archives.gov/cui/registry/limited-dissemination.

Example 1

If you want to apply the Accident Investigation category to your document, you want to see the CUI//SP-AIV marking in the header and footer.

The SP indicates it is a specified CUI.

To apply the CUI//SP-AIV marking:

  1. From the ribbon in your Word document, click Select.

  2. From the Controlled section, click Yes.

  3. From the Contains CUI section, click Yes.

  4. From the Organizational Grouping, select Full List.

  5. From the CUI Categories section, click Accident Investigation.

  6. Click OK.

Example 2

If you want to send an email to Federal Employees Only, you want to apply that Dissemintation Control. The recipient will see the CUI//FED ONLY marking in the body of the email.

To apply the CUI//FED ONLY marking:

  1. From the ribbon in your email message, click Select.

  2. From the Controlled section, click Yes.

  3. From the Contains CUI section, click Yes.

  4. From the DISSEMINATION CONTROL section, click Federal Employees only.

  5. Click OK.

Example 3

You can apply both a CUI Category and Dissemination Control to your Word document or email. In this example, you will apply the Legislative Materials and Physical Security CUI categories for Federal Employees and Contractors only. Your header should appear as CUI//LMI/PHYS/FEDCON.

To apply this marking:

  1. From the ribbon in your email message, click Select.

  2. From the Controlled section, click Yes.

  3. From the Contains CUI section, click Yes.

  4. From the CUI Category, select Legislative Materials (LMI) and Physical Security (PHYS).

  5. From the DISSEMINATION CONTROL section, click Federal Employees and Contractors Only.
